How it works 01 / 03
1 APPS & USERS

Your transaction starts here.

Wallets, dApps and DeFi submit transactions to Ternus exactly like they would to Ethereum — same Solidity, same tooling, no rewrite.

2 TERNUS · LAYER 2

Executed off-chain, rolled into one batch.

Ternus runs thousands of transactions per second off-chain, then compresses them into a single proof — fast and near-zero cost.

3 ETHEREUM · LAYER 1

Settled and secured on mainnet.

The proof is verified on Ethereum L1 — your transaction inherits mainnet-grade security, with none of the congestion.

Build for developers

Point your RPC at Ternus, ship in minutes.

Fully EVM-equivalent. Your existing Foundry / Hardhat workflow just works — no new SDK, no rewrite. Three steps, same as mainnet.

01
Add the network
One RPC URL + chain ID.
02
Deploy your contract

03
Go live

# Ternus testnet — add to foundry.toml [rpc_endpoints] ternus = "https://rpc.testnet.ternus.xyz" # chain id 9400 # currency ETH # explorer https://scan.testnet.ternus.xyz
# deploy with foundry — same as mainnet $ forge create src/Token.sol:Token \ --rpc-url ternus \ --private-key $PK \ --broadcast [⠊] Compiling... no files changed Deployer: 0x9f3c…a21b Deployed: 0x4eD8…77Fc ✓ settled to Ethereum L1 in 1.2s
